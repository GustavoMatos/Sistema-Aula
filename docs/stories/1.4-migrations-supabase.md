## Schema Resumido

```
workspaces (id, name, settings, created_at, updated_at)
    │
    ├── users (id→auth.users, workspace_id, email, full_name, role)
    ├── whatsapp_instances (id, workspace_id, instance_name, status, ...)
    ├── kanban_stages (id, workspace_id, name, color, position, is_final)
    │       │
    │       └── followup_configs (id, stage_id, delay_hours, message_text, ...)
    │
    └── leads (id, workspace_id, stage_id, name, phone, tags[], ...)
            │
            ├── messages (id, lead_id, direction, content, status, ...)
            ├── scheduled_messages (id, lead_id, followup_config_id, scheduled_for, ...)
            └── lead_history (id, lead_id, action, from_stage_id, to_stage_id, ...)
```

**Índices importantes:**
- `idx_leads_workspace_id` - Filtro principal
- `idx_leads_stage_id` - Filtro por estágio
- `idx_leads_phone` - Busca por telefone
- `idx_scheduled_messages_status` - Fila de processamento

**RLS Helper Function:**
```sql
CREATE OR REPLACE FUNCTION get_user_workspace_id()
RETURNS UUID AS $$
  SELECT workspace_id FROM users WHERE id = auth.uid()
$$ LANGUAGE SQL SECURITY DEFINER STABLE;
```
